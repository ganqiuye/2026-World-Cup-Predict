# 综合权重算法（v2）

**时区标准：** 全部以 `Asia/Shanghai`（北京时间 CST）为准。  
**数据时效：** `data_as_of` 使用 ISO8601 并标注 `(CST)`。

---

## 1. 总公式

对单场比分 `(h, a)` 的联合概率：

```
P(h,a) = Z⁻¹ × exp( Σᵢ wᵢ × Sᵢ(h,a) )
```

其中 `Z` 为归一化常数，`Sᵢ` 为第 i 维得分函数（越高表示该比分越可能）。

**六维权重（默认，和为 1.0）：**

| 维度 | 符号 | 默认权重 wᵢ | 说明 |
|------|------|-------------|------|
| 纯实力 | S_str | **0.32** | Elo/xG/近期状态 |
| 博弈修正 | S_gt  | **0.24** | 情景 A–L |
| 同组赛序 | S_seq | **0.14** | 情景 K：先赛后赛传导 |
| 淘汰赛路径 | S_path | **0.12** | 情景 J：32 强对手强弱 |
| 球员个人 | S_pl  | **0.10** | 情景 L：金靴/最佳球员 |
| 战术/外部 | S_ext | **0.08** | 气候、伤病、裁判、赛程密度 |

**主导因素判定：**

```
dominant = argmax( wᵢ × Var(Sᵢ) )
```

若 `w_gt×S_gt` 与 `w_str×S_str` 差值 < 5%，标为 `[均衡]`。

## 2. 各维度得分函数

### 2.1 纯实力 S_str

基于 Poisson 进球模型：

```
λ_home = base_xG_home × home_adv × form_home
λ_away = base_xG_away × form_away

P_poisson(h,a) = Poisson(h;λ_h) × Poisson(a;λ_a)
S_str(h,a) = ln P_poisson(h,a)
```

- `base_xG` 来自近 10 场加权 xG（友谊赛 ×0.5，近 6 月 ×1.3）
- Elo 差 Δ → 胜率基准：Δ=100 约等于 64% 主胜
- 输出 **纯实力最可能比分** = argmax P_poisson

### 2.2 博弈修正 S_gt

对触发的情景 A–I，对比分施加对数几率偏移：

| 情景 | 偏移示例 |
|------|----------|
| A 生死战 | 大比分、高进球比分 +0.15~0.25 |
| B 打平出线 | 平局 (h=a) +0.20~0.35 |
| C 已出线/淘汰 | 强队进球 λ ×0.75~0.85 |
| E 战略性控分 | 小负/平局 +0.25 |
| F 同轮协同 | 见 2.3 与 F 联动 |
| G 净胜球算计 | 领先后大比分 −0.20 |
| H 默契球 | 平局 +0.30 |
| I 轮换 | 冷门比分 +0.15 |

多情景叠加时，单维偏移上限 ±0.15（对数几率）。

### 2.3 同组赛序 S_seq（情景 K）

**适用：** 同组内存在**已结束**的本轮或前轮比赛，且结果改变本场出线集合时。

**算法：**

1. 按北京时间排序同组本轮场次，标记 `order_index`
2. 对已赛场次 `m₁…mₖ`，更新临时积分榜 `standings'`
3. 对当前场 `(h,a)`，计算在 `standings'` 下各可能比分的出线状态集合 `Q(h,a)`
4. 战意系数：

```
motivation(team, Q) = 1.0                    若 Q 中出线不变
                    = 1.0 + 0.12×must_win    若必须赢才能出线
                    = 1.0 − 0.10×already_in  若已确保名次
                    = 1.0 + 0.08×rival_effect 若先赛结果改变对手策略
```

5. 调整 λ：

```
λ_home' = λ_home × motivation(home, Q)
λ_away' = λ_away × motivation(away, Q)
S_seq(h,a) = ln Poisson(h;λ_h') + ln Poisson(a;λ_a') − S_str(h,a)
```

### 4.3 实时触发（动态 K）

| 事件 | 动作 |
|------|------|
| 先赛开球前 | 用先赛 Top-N 比分边际预测后赛 |
| 先赛终场 | **强制**用 `actual_score` 重算后赛 |
| R3 同轮 12 场 | **关闭 K**，启用 **F + Q** |

完整算法见 [calibration.md](calibration.md) §4。

### 2.4 淘汰赛路径 S_path（情景 J）

对每组可能名次 `r ∈ {1,2,3}`，查 `knockout-bracket.json` 得 32 强对手 `O(r)`，估算胜率 `p(r)`。

```
path_value(r) = p(r) − p(r_alt)
```

若 `(h,a)` 导致名次 r 且 `path_value(r) < −0.08`（第二路径更优），则降低赢球比分权重、提高控分比分权重。

### 2.5 球员个人 S_pl（情景 L）

维护 `data/players.json` 中候选球员列表。

对球员 `p`：

```
award_pressure(p) = w_boot × rank_boot(p) + w_ball × rank_ball(p)
```

- `rank_boot`：射手榜排名倒数归一化（第 1 名 = 1.0）
- `rank_ball`：最佳球员竞争热度（0~1，人工/新闻标注）

球队进攻 λ 修正：

```
λ_team += Σ_p ( award_pressure(p) × minutes_share(p) × 0.08 )
```

个人刷数据倾向 → 增加 2–3 球、主力进球比分权重；已锁定出线 + 个人目标已达成 → 轮换 −0.10。

**2026 窗口重点关注：**

| 球员 | 球队 | 压力来源 |
|------|------|----------|
| 梅西 | 阿根廷 | 历史射手王已达成，战意↓ |
| 姆巴佩 | 法国 | 金靴竞争（4 球），末轮对挪威 |
| 哈兰德 | 挪威 | 金靴竞争（4 球） |
| C罗/拉莫斯等 | 葡萄牙 | 末届世界杯个人数据 |

### 2.6 战术/外部 S_ext

- 高温高湿：体能型球队 λ −3%
- 少休 1 天：λ −3%
- 核心伤停：λ −8~15%
- 裁判严哨：黄牌多、进球略 −2%

## 3. 合成与输出

### 3.1 比分排序

```
score_total(h,a) = Σᵢ wᵢ × Sᵢ(h,a)
```

取 Top 8 归一化为 `score_distribution`（百分比和 ≈ 100%）。

### 3.2 必输出字段

```json
{
  "weight_breakdown": {
    "strength": 0.32,
    "game_theory": 0.24,
    "sequence": 0.14,
    "path": 0.12,
    "player": 0.10,
    "external": 0.08
  },
  "weight_contributions": {
    "strength_pct": 38,
    "game_theory_pct": 22,
    "sequence_pct": 18,
    "path_pct": 8,
    "player_pct": 9,
    "external_pct": 5
  }
}
```

`weight_contributions` 为**该场实际**各维对最可能比分的贡献占比（可因情景叠加而变化）。

### 3.3 置信度

| 星级 | 条件 |
|------|------|
| 5 | 最可能比分概率 > 22% 且单一维度主导 |
| 4 | 18–22% |
| 3 | 14–18% |
| 2 | 10–14% |
| 1 | < 10% 或多情景严重冲突 |

---

## 4. 计算顺序（强制）

```
1. 联网（中国源）→ 更新 schedule / standings / players
2. 按北京时间筛选窗口内场次
3. 同组内按开赛时间排序 → 应用 K（已赛结果锁定后再预测未赛）
4. 逐步：S_str → S_gt → S_seq → S_path → S_pl → S_ext
5. 合成 → 写 predictions.json → inject 仪表盘
```
